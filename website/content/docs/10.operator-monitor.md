# Operator & Monitor

Le système **Operator** est le centre de contrôle de l'expérience Rift. Il permet de configurer les préréglages visuels et de sélectionner les flux vidéo qui seront diffusés dans le **Monitor**.

## Fonctionnement

L'expérience se déroule en plusieurs étapes fluides :

1.  **Sélection du Mode** : Au démarrage, l'opérateur choisit entre deux modes d'ambiance :
    *   **Dream** : Une esthétique plus calme et éthérée (Bleu).
    *   **Nightmare** : Une ambiance plus sombre et intense (Rouge/Noir).

2.  **Configuration des Caméras** :
    *   L'opérateur voit une grille de toutes les webcams connectées au système.
    *   Il sélectionne les flux spécifiques qui doivent être actifs pour la session.

3.  **Standby** :
    *   Une fois la configuration terminée, le système passe en état d'attente (**Standby**).
    *   L'interface reste figée sur cet écran jusqu'à ce que le signal de démarrage (`operator_start_system`) soit reçu via WebSocket. Cela permet de synchroniser parfaitement le lancement visuel avec le début physique ou narratif de l'expérience.

4.  **Monitor** :
    *   C'est la vue finale, souvent projetée ou affichée sur un grand écran.
    *   Elle affiche les flux caméras sélectionnés dans une grille dynamique style "Matrix".
    *   C'est la fenêtre visuelle sur le "Rift".

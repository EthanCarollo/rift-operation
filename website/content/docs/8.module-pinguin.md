# Module Pinguin

Le module **Pinguin** est une expérience interactive où l'objectif principal est de découvrir des informations cachées à travers une conversation naturelle avec l'IA.

## L'Objectif : Trouver Paul

Le but ultime de l'interaction avec le module Pinguin est de **trouver le mot "Paul"**.

Les joueurs doivent poser des questions, enquêter et recouper les informations fournies par le pinguin pour remonter la piste jusqu'à ce nom. C'est une chasse à l'information sémantique.

Certaines de ces fiches contiennent déjà toutes les informations nécessaires. Elles servent de base de vérité pour le système de Q&A (Question & Answer). Le défi pour le joueur est d'extraire cette information qui est "sous ses yeux" mais accessible uniquement via le dialogue.

## Système Q&A

Le système interroge cette base de connaissance statique pour répondre aux joueurs.

![Q&A Pinguin](/pinguin/q&a-pinguin.png)

Cela permet une interaction fluide où le pinguin agit comme un gardien de l'information, ne révélant les secrets (comme le lien vers Paul) que si les bonnes questions sont posées.

## Architecture Technique

Ce module repose sur l'intégration de deux briques technologiques majeures documentées par ailleurs :

1.  **Transcription Temps-Réel (STT)** : Le module s'appuie sur [Kyutai Swift](/docs/6.kyutai-swift) pour transformer la voix du joueur en texte de manière instantanée.
2.  **Intelligence Sémantique (QA)** : L'analyse des questions et la recherche des réponses sont gérées par le module [Pinguin QA](/docs/7.pinguin-qa), qui utilise la recherche vectorielle pour comprendre le sens des demandes.

L'orchestration de ces deux services permet de créer une boucle de feedback rapide (Audio -> Texte -> Recherche -> Audio/Réponse) essentielle pour l'immersion.
